Where my imagined flow differs from your spec

  • Five pages, no skips. Mine marches through all four blueprint inputs for everyone. Yours is leaner — 2–3 pages with the conditional Level skip. Yours respects drop-off risk more.
  • Numbers up front. Mine asks for current load + threshold anchor (FTP, CSS, T-pace, 2k split, 1RMs) on day one so zones exist immediately. Yours defers to a PR or a 2-week observed baseline — friendlier to new users.
  • Timeline and injuries. Mine collects weeks-to-race and injury flags (blueprint inputs 3 and 4). Yours leaves them out of pages 1–3 — they would need a home later in the flow.
  • No training-focus fork. Mine assumes everyone is event-driven, as the docs do. Yours adds Fat loss and Learn-the-basics paths the blueprints never mention — a real product insight the docs miss.
  • Tier handling agrees. Neither shows a beginner/intermediate/advanced label. The docs derive level from load numbers; your spec derives it from PRs. Both kill the current self-labeled mode screen.
